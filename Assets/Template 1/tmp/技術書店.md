# urp12からurp14へのアップグレード

## はじめに

urp10からurp12への移行は、SwapBuffer程度でしたのでそこまで対応に時間がかかりませんでした。<br>
また、基本的な自作RendererFeatureの動作にはあまり影響はありませんでした。<br>

しかし今回のurp14では、アップデートすると自作RendererFeatureが動かないものが多数あったので取り上げさせて頂きました。

業務ではグラフィックス周りを横断的に扱うケースもあり、urp12でもurp14でも動くように改修していきます。


## 環境

### 移行前

* Unity : 2021.3.15f1
* Universal RP : 12.1.8

### 移行後

* Unity : 2022.3.20f1
* Universal RP : 14.0.10

## なぜ動かなくなってしまうのか

URP14のWhat's newを見ると、以下のように記載されています。

> Full screen draws in URP now use the SRP Core Bliter API<br>
All the calls to cmd.Blit method are replaced with the Blitter API.<br>
This ensures a correct and consistent way to perform full screen draws.<br>
In the current URP version, using cmd.Blit might implicitly enable or disable XR shader keywords, which breaks XR SPI rendering.<br>
Refer to the Perform a full screen blit in URP page to read how to use the Blitter API.


要約すると、以下です。

* URPのポストエフェクトでは、SRP Core Bliter APIというものを使用するようになった
* 今までのcmd.BlitメソッドはBliter APIに置き換えられるが、XR環境では壊れるかもしれない

つまり、今まで`cmd.Blit`で呼んでいたものが`Bliter`に置き換わったことで動かなくなってしまっています。

## 対応するべき？

このBliterAPIを使用することにより、`RenderTargetHandle`が`RTHandle`へとなります。<br>
RTHandleは複数解像度のカメラ間でRenderTextureをよしなに使い回してくれる機能です。<br>
なので、基本的に対応した方が余計なRenderTextureを使用することがないので良いです。

また、このurp14のアップデートで`RenderTargetHandle`等の一部メソッドが`obsolete`となっているため、いずれは対応しないといけないです。

## urp14に移行する

上記のコードをそのままurp14の環境へ移行してみます。<br>
すると、描画されないどころかコンパイルエラーが出るので1つずつ対応します。

### コンパイルエラー

```
You can only call cameraColorTarget inside the scope of a ScriptableRenderPass. Otherwise the pipeline camera target texture might have not been created or might have already been disposed.
```
ColorTargetが既に破棄されてる可能性がある旨のエラーが出ています。<br>
このエラー通りに`ScriptableRenderPass`のスコープで呼ぶようにします。<br>
また、`cameraColorTarget`は`obsolete`なため、`cameraColorTargetHandle`に置換します。

```cs
    // GrayScaleRendererFeature.cs
    ...

    public override void AddRenderPasses(ScriptableRenderer renderer, ref RenderingData renderingData)
    {
#if !UNITY_2022_2_OR_NEWER
        grayscalePass.SetRenderTarget(renderer.cameraColorTarget);
#endif
        renderer.EnqueuePass(grayscalePass);
    }

#if UNITY_2022_2_OR_NEWER
    // urp14で新しく追加されたメソッドでカラーバッファを渡すようにする
    public override void SetupRenderPasses(ScriptableRenderer renderer, in RenderingData renderingData)
    {
        grayscalePass.SetRenderTarget(renderer.cameraColorTargetHandle);
    }
#endif
```

コンパイルエラーは出なくなり、画面にグレースケールがかかります。<br>
ですが、urp14の本題であるRTHandleに触れていないのと警告が出ているので修正します。

### 警告を取り除く

```
'RenderTargetHandle' is obsolete: 'Deprecated in favor of RTHandle'
```

冒頭でも挙げた、`RenderTargetHandle`を`RTHandle`に変更する旨の警告が出ています。<br>
まずは、`RenderTargetHandle`周りの処理をディレクティブで括ってしまいます。<br>
その後、RendererFeatureから渡ってきたカラーバッファを`RTHandle`型で受け取るようにします。

```cs
    // GrayscalePass
    ...

#if UNITY_2022_2_OR_NEWER
    // urp14からはRTHandleを使うように
    private RTHandle tmpRenderTargetHandle;
    private RTHandle cameraColorTarget;
#else
    private RenderTargetHandle tmpRenderTargetHandle;
    private RenderTargetIdentifier cameraColorTarget;
#endif

    public GrayscalePass(Shader shader)
    {
        material = CoreUtils.CreateEngineMaterial(shader);
        renderPassEvent = RenderPassEvent.AfterRenderingTransparents;
#if !UNITY_2022_2_OR_NEWER
        tmpRenderTargetHandle.Init("_TempRT");
#endif
    }

#if UNITY_2022_2_OR_NEWER
    // RTHandleとして受け取る
    public void SetRenderTarget(RTHandle target)
#else
    public void SetRenderTarget(RenderTargetIdentifier target)
#endif
    {
        cameraColorTarget = target;
    }
```

次にExecute関数内の対応です。<br>
urp12では、`cmd.GetTemporaryRT`でRenderTextureを確保していましたが、urp14では`RenderingUtils.ReAllocateIfNeeded`でRenderTextureを確保します。<br>
今回の例では、最低限の引数で行っていますが`FilterMode`や`TextureWrapMode`,RenderTextureの名前等も指定することができます。<br>
その確保したRenderTextureを元に、Blit()を呼び出してポストエフェクトをかけます。


```cs
// GrayscalePass
...

public override void Execute(ScriptableRenderContext context, ref RenderingData renderingData)
{
    ...

    using (new ProfilingScope(cmd, profilingSampler))
    {
#if UNITY_2022_2_OR_NEWER
        // RenderingUtils.ReAllocateIfNeededでRenderTextureを確保する
        RenderingUtils.ReAllocateIfNeeded(ref tmpRenderTargetHandle, descriptor);
        Blit(cmd, cameraColorTarget, tmpRenderTargetHandle, material);
        Blit(cmd, tmpRenderTargetHandle, cameraColorTarget);
        // MEMO : この例だとBlit(cmd, ref renderingData, material);の方が好ましい
#else
        cmd.GetTemporaryRT(tmpRenderTargetHandle.id, descriptor);
        cmd.Blit(cameraColorTarget, tmpRenderTargetHandle.Identifier(), material);
        cmd.Blit(tmpRenderTargetHandle.Identifier(), cameraColorTarget);
        cmd.ReleaseTemporaryRT(tmpRenderTargetHandle.id);
#endif
    }
    ...
}
```

#### Blitの中身

Execute関数内だけでは、今までとそこまで変わりませんがBlit関数の中身を見てみるとBliterAPIが使用されているのがわかります。<br>

```cs
public void Blit(CommandBuffer cmd, RTHandle source, RTHandle destination, Material material = null, int passIndex = 0)
{
    if (material == null)
        Blitter.BlitCameraTexture(cmd, source, destination, bilinear: source.rt.filterMode == FilterMode.Bilinear);
    else
        Blitter.BlitCameraTexture(cmd, source, destination, material, passIndex);
}
```
